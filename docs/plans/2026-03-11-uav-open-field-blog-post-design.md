# 无人机仿真场景博客设计

## 目标

在博客中新增一篇面向中文读者的技术文章，介绍如何基于 `ROS 2 Humble + ros_gz + Gazebo` 搭建一个可运行的无人机仿真场景，并把我这次实际落地的目录结构、关键文件、运行命令和踩坑记录整理出来。

## 读者假设

- 已经安装好 `ROS 2 Humble`
- 已经安装好可用的 Gazebo / `ros_gz`
- 希望快速理解这个无人机场景的组成方式，并能直接复现

文章不重复环境安装细节，而是简要说明环境要求，并链接站内现有安装文章。

## 文章定位

采用“原理说明 + 实操步骤结合”的写法。

核心要求：

- 先解释这个场景是怎么组织起来的
- 再给出从 clone 到运行的完整步骤
- 最后补充真实踩坑和排障信息

这样既适合第一次复现，也适合后续自己改模型、改话题、改场景时回查。

## 建议标题

`ROS 2 Humble + Gazebo 搭建无人机仿真场景实战：从空旷起降场到 ROS2 话题桥接`

## 建议摘要

记录我如何在 Ubuntu 22.04 上，基于 ROS 2 Humble、ros_gz 和本地四旋翼模型，搭建一个可直接运行的无人机仿真场景，并整理出完整目录结构、关键文件说明、运行命令和常见问题。

## 文章结构

### 1. 适用范围

说明本文面向已经装好 `ROS 2 Humble` 与 Gazebo 的读者，并放出两篇站内安装文链接。

### 2. 这次搭的是什么场景

说明目标是一个空旷起降场：

- 本地四旋翼模型
- 空旷 world
- IMU / GPS / Camera / Odometry
- ROS 2 与 Gazebo bridge
- 基础控制入口

### 3. 工作区结构

展示 `~/uav_sim_ws` 的目录树，并说明各目录职责。

### 4. 核心文件说明

按文件讲清架构：

- `worlds/open_field.sdf`
- `models/quadrotor/model.sdf`
- `config/bridge.yaml`
- `launch/open_field_uav.launch.py`
- `scripts/takeoff_demo.py`

### 5. 实际搭建步骤

给出：

- 获取项目
- 安装依赖
- `colcon build`
- `ros2 launch`
- `ros2 run takeoff_demo.py`
- `ros2 topic list` 验证

### 6. 真实踩坑记录

重点写这次调试中真实出现的问题：

- `Link ... could not be found`
- `allocation matrix rank is 3`
- `ros2 run` 找不到脚本可执行入口

并解释为什么会出现。

### 7. 别人怎么复用

给出 GitHub 仓库地址和最短复现命令。

## 风格要求

- 对齐现有安装文章的中文技术博客风格
- 命令块完整，能直接复制
- 少写空泛背景，多写真实结构和踩坑信息
- 不把文章写成单纯的流水账提交记录

## 非目标

这篇文章不展开：

- PX4 / ArduPilot
- 多无人机场景
- 避障、SLAM、路径规划算法
- 更复杂的城市 / 室内场景设计

## 验收标准

完成后应满足：

1. 文章出现在 `content/blog/` 下并能被博客识别
2. front matter 与现有文章格式一致
3. 文章内包含站内安装文链接
4. 文章内包含目录结构、关键文件说明、运行步骤和踩坑说明
5. 内容可以直接支撑别人按文复现该场景
